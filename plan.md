# План проекта — RAG-подбор товаров

Хочу сделать сервис, в который пишешь запрос на человеческом языке («ищу тёплую куртку до 15к для города»), а он возвращает подборку подходящих товаров из каталога и короткое объяснение, почему выбрал именно их.

## Что буду использовать

- Бэкенд — FastAPI.
- Фронт — Streamlit (одна страница: поле ввода, фильтры, карточки товаров, ответ модели).
- База — PostgreSQL с расширением pgvector, чтобы искать по эмбеддингам прямо в SQL.
- Эмбеддинги — локально через sentence-transformers (multilingual-e5-small, потому что поддерживает русский и маленькая).
- LLM — локально через Ollama (начну с llama3.1:8b или qwen2.5:7b, выберу какая лучше на русском).
- Всё поднимается одной командой через docker-compose: postgres, backend, frontend, ollama.

## Данные

Возьму открытый датасет товаров одежды (или дамп каталога, если пойму, что можно переиспользовать). Основные поля: название, описание, категория, цена, цвет, материал, пол, картинка, ссылка. Один раз прогоняю через эмбеддер и кладу векторы в колонку `embedding` у товара.

## Как устроен поиск

1. Пользователь пишет запрос в Streamlit.
2. Фронт зовёт `POST /search` у бэкенда.
3. Бэкенд считает эмбеддинг запроса, делает векторный поиск в postgres с фильтрами (цена, категория, пол), берёт топ-8.
4. Эти 8 товаров + запрос уходят в локальную LLM через Ollama с промптом «выбери 3 самых подходящих и объясни почему».
5. Ответ (товары + текст модели) возвращается на фронт.

## Эндпоинты

- `GET /health` — для healthcheck докера.
- `POST /search` — основной, принимает запрос и фильтры, возвращает товары и ответ LLM.
- `GET /product/{id}` — карточка товара.
- `GET /categories` — список категорий для фильтров на фронте.
- `POST /feedback` — палец вверх/вниз на ответ, пишется в БД.

## Таблицы в БД

- `products` — каталог с колонкой embedding vector(384).
- `queries` — что спрашивал пользователь и что ответила модель.
- `feedback` — оценки ответов.

## Этапы работы

1. Сделать скелет репо: структура папок, docker-compose, Dockerfile для бэка и фронта, `.env.example`, README.
2. Поднять FastAPI с `/health` в докере, убедиться что всё стартует.
3. Добавить postgres+pgvector в compose, подключить SQLAlchemy, накатить модели через alembic.
4. Написать скрипт импорта каталога из JSON в БД.
5. Подключить sentence-transformers, посчитать эмбеддинги для всех товаров, построить ivfflat индекс.
6. Сделать `/search` без LLM — только векторный поиск с фильтрами.
7. Добавить Ollama в compose, подключить LLM к `/search`, написать нормальный промпт.
8. Сверху прикрутить Streamlit: поле поиска, фильтры, карточки, кнопки feedback.
9. Написать тесты на эндпоинты (LLM в тестах замокаю).
10. Докрутить README: как запускать, архитектура, скриншоты.

## Что должно получиться в итоге

Репозиторий, который клонируется, поднимается одной командой `docker compose up`, открывается `localhost:8501`, пишешь запрос — получаешь релевантные товары с объяснением от локальной модели. Всё работает офлайн, без платных API.
